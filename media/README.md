## Guidelines

**Screenshots should show:**
- Clean, professional UI
- Real queries with cited answers (redact any sensitive content)
- Audit trail in action
- Role-based access differences

**Screenshots should NOT show:**
- Terminal/console output
- Error messages or debugging views
- Internal configuration panels
- System architecture diagrams with technical details
- Library names, version numbers, or dependency lists

**Videos should be:**
- 30–90 seconds each
- No voiceover revealing technical details
- Professional — think Apple product demo, not developer tutorial
- Focused on the user experience, not the technology

**Benchmark charts should show:**
- Results and outcomes only
- No methodology details that reveal architecture
- Professional design consistent with brand
